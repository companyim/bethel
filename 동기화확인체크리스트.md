# Firebase 동기화 확인 체크리스트

## 1. Firestore 데이터베이스 생성 확인
- [ ] Firebase 콘솔 (https://console.firebase.google.com/) 접속
- [ ] 프로젝트 "bethel-attendance" 선택
- [ ] 왼쪽 메뉴에서 "Firestore Database" 클릭
- [ ] 데이터베이스가 생성되어 있는지 확인
- [ ] 데이터베이스가 없으면 "데이터베이스 만들기" 클릭하여 생성

## 2. 보안 규칙 확인
- [ ] Firestore Database > 규칙 탭으로 이동
- [ ] 다음 규칙이 설정되어 있는지 확인:

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /attendanceApp/{document} {
      allow read, write: if true;
    }
  }
}
```

- [ ] "게시" 버튼 클릭하여 규칙 적용

## 3. 브라우저 콘솔 확인
- [ ] 개발 서버 실행: `npm run dev`
- [ ] 브라우저에서 `http://localhost:5173` 접속
- [ ] F12로 개발자 도구 열기
- [ ] 콘솔 탭에서 다음 메시지 확인:
  - ✅ "Firebase가 초기화되었습니다"
  - ❌ 오류 메시지가 있는지 확인

## 4. 데이터 동기화 테스트

### 테스트 1: Firebase에 데이터 저장 확인
1. 학생 추가
2. 브라우저 콘솔에서 "Firebase에 students 저장 완료" 메시지 확인
3. Firebase 콘솔 > Firestore Database > 데이터 탭에서 확인
   - `attendanceApp` 컬렉션이 생성되었는지 확인
   - `students` 문서가 생성되었는지 확인

### 테스트 2: 다른 기기에서 확인
1. 모바일 브라우저에서 동일한 URL 접속
2. 관리자 로그인
3. 학생 목록이 동기화되었는지 확인

## 5. 일반적인 오류 및 해결 방법

### 오류: "Permission denied" 또는 "Missing or insufficient permissions"
**원인**: Firestore 보안 규칙이 잘못 설정됨
**해결**: 보안 규칙을 위의 규칙으로 변경하고 "게시" 클릭

### 오류: "Firebase가 초기화되지 않았습니다"
**원인**: Firebase 설정 파일에 문제가 있음
**해결**: `src/config/firebase-config.js` 파일 확인

### 데이터가 동기화되지 않음
**확인 사항**:
1. Firestore 데이터베이스가 생성되었는지 확인
2. 보안 규칙이 올바르게 설정되었는지 확인
3. 브라우저 콘솔에서 오류 메시지 확인
4. Firebase 콘솔에서 데이터가 저장되었는지 확인

## 6. 데이터 초기화 (필요시)

Firebase 데이터를 초기화하려면:
1. Firebase 콘솔 > Firestore Database > 데이터 탭
2. `attendanceApp` 컬렉션 삭제
3. 페이지 새로고침
