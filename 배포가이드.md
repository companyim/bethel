# GitHub Pages 배포 가이드

## 배포 방법

### 방법 1: GitHub Desktop 또는 웹 인터페이스 사용

1. **GitHub 저장소 확인**
   - 저장소 URL: `https://github.com/[사용자명]/[저장소명]`
   - 또는 GitHub에서 새 저장소 생성

2. **dist 폴더 내용 업로드**
   - `c:\cursor\dist` 폴더의 모든 파일 선택
   - GitHub 저장소의 루트(/) 또는 `docs` 폴더에 업로드
   - 또는 `gh-pages` 브랜치 생성 후 업로드

3. **GitHub Pages 설정**
   - 저장소 페이지로 이동
   - Settings > Pages 메뉴 클릭
   - Source 선택:
     - 루트에 업로드한 경우: "Deploy from a branch" > Branch: `main` > Folder: `/` > Save
     - docs 폴더에 업로드한 경우: "Deploy from a branch" > Branch: `main` > Folder: `/docs` > Save
     - gh-pages 브랜치 사용한 경우: "Deploy from a branch" > Branch: `gh-pages` > Folder: `/` > Save

4. **배포 완료**
   - 몇 분 후 `https://[사용자명].github.io/[저장소명]` 접속 확인

### 방법 2: Git 명령어 사용 (Git 설치 필요)

1. **Git 설치**
   - https://git-scm.com/download/win 에서 다운로드 및 설치

2. **저장소 초기화 및 배포**
```bash
cd c:\cursor
git init
git add dist
git commit -m "Deploy to GitHub Pages"
git branch -M main
git remote add origin https://github.com/[사용자명]/[저장소명].git
git subtree push --prefix dist origin gh-pages
```

### 방법 3: GitHub Actions 사용 (자동 배포)

`.github/workflows/deploy.yml` 파일 생성:

```yaml
name: Deploy to GitHub Pages

on:
  push:
    branches:
      - main

jobs:
  build-and-deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      
      - name: Setup Node.js
        uses: actions/setup-node@v3
        with:
          node-version: '18'
      
      - name: Install dependencies
        run: npm install
      
      - name: Build
        run: npm run build
      
      - name: Deploy to GitHub Pages
        uses: peaceiris/actions-gh-pages@v3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./dist
```

## 주의사항

1. **Firebase 설정 확인**
   - `src/config/firebase-config.js` 파일이 빌드에 포함되는지 확인
   - Firebase 설정이 올바른지 확인

2. **Base 경로**
   - 현재 `vite.config.js`에서 `base: './'`로 설정되어 있음
   - 저장소 루트에 배포하면 문제 없음
   - 하위 폴더에 배포하려면 `base: '/[저장소명]/'`로 변경 필요

3. **배포 후 확인**
   - 브라우저 콘솔(F12)에서 Firebase 초기화 메시지 확인
   - 다른 기기에서 접속하여 동기화 확인

## 문제 해결

- **404 오류**: Base 경로 설정 확인
- **Firebase 오류**: Firebase 설정 파일 확인
- **스타일 깨짐**: CSS 파일 경로 확인
